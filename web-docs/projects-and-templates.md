# 项目与模板

项目库和模板库是日程补全的基础。项目决定关联范围，模板决定自动补全时可以填什么内容。

## 项目库

项目可以来自 Excel 的 `项目清单`，也可以手动新增。导入项目后仍然可以编辑，手动编辑过的项目会作为本地维护内容保留。

项目类别只保留 `总部项目`、`公司项目`、`院控项目`、`探索项目`。项目号包含 `KF`、`KY` 时会归为公司项目，包含 `NM` 时会归为院控项目。团队、负责人或其他说明应写在备注中，不作为类别。

## 模板库

模板可以来自 Excel 的 `一级和二级菜单`、2026 年及以后的历史填报记录，也可以来自旧 JSON 配置或手动新增。`Sheet2` 作为旧模板来源不会导入。模板包含工作性质、工作类别、关联项目、工作形式、备注和权重。

权重只影响自动补全时被选中的概率，不代表工时长度。工时长度由空白时间段和补全规则决定。

## 常用维护方式

1. 先导入 Excel，得到项目、模板和 2026 年后的填报历史。
2. 把常用项目设为准确类别，团队等说明放在备注中。
3. 删除不会再使用的模板，编辑保留的模板。
4. 为固定安排设置固定模板，为随机补全保留通用模板。

## 旧记录兼容

旧 Excel 中的 `内容属性` 会按 `关联项目` 处理。旧记录中的历史项目不会自动进入项目库，项目库优先来自 2026 年后的有效配置和 `项目清单`。
