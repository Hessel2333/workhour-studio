# 菜单规则

工时内容按 `工作性质 → 工作类别 → 关联项目 / 工作形式` 的顺序选择。后面的选项会根据前面的选择收窄，避免填出平台无法接受或统计口径不清的组合。

## 工作性质与类别

`科研工作` 下可选 `总部项目`、`公司项目`、`院控项目`、`创新创效`、`探索项目`、`其他科研生产`。

`事务性工作` 下可选 `实验室日常维护`、`会议`、`其他事务性`、`出差`。

`请假` 下只保留 `请假`。

## 工作形式

`测试实验`、`合成实验`、`样品寄送`、`物资采购` 只出现在科研工作中。

会议、培训、调研、文字撰写等通用工作形式会按工作性质保留。切换工作性质后，如果原工作形式不再适用，系统会自动切换到当前范围内的第一个可用项。

## 关联项目

关联项目跟随工作类别过滤。选择 `探索项目` 时，只显示探索项目池里的项目；选择 `总部项目`、`公司项目`、`院控项目`、`创新创效` 时，只显示同类别项目。

没有明确关联项目的记录，可以选择 `备注 / 不关联项目`，再在备注中写清工作内容。

## 切换时的自动修正

当工作性质或工作类别变化时，系统会检查已选类别、项目和形式是否仍然有效。无效的类别或形式会自动修正；不属于当前类别的项目会被清空或保留为历史值，便于检查旧记录。
