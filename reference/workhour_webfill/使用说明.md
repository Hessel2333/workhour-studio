# 🚀 科技管理平台工时填报自动填写助手 - 完整版

## 📋 功能特点

✅ **一键自动填写** - 智能填充所有必要字段，支持多行工时填写  
✅ **智能日期时间** - 自动填入当前日期和正确的时间格式  
✅ **多配置管理** - 支持创建、编辑、导入导出多套配置方案  
✅ **项目自动选择** - 根据关键词自动搜索并选择关联项目  
✅ **共同完成人自动选择** - 自动搜索并选择指定的共同完成人  
✅ **工作性质类别联动** - 根据工作性质自动匹配对应的工作类别  
✅ **智能表格识别** - 动态检测页面元素，适应不同页面结构  
✅ **界面位置优化** - 按钮位置调整到左下角，操作更便捷  
✅ **随机工作内容** - 从可自定义的模板库中随机选择工作内容

## 🛠️ 安装方法

### 步骤1：安装浏览器扩展
选择以下任一浏览器扩展：
- **Chrome/Edge**: [Tampermonkey](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo)
- **Firefox**: [Greasemonkey](https://addons.mozilla.org/en-US/firefox/addon/greasemonkey/)

### 步骤2：安装脚本
1. 安装完扩展后，点击浏览器右上角的扩展图标
2. 选择"添加新脚本"
3. 删除默认内容，复制粘贴 `工时自动填写脚本.js` 中的所有代码
4. 按 `Ctrl+S` 保存

### 步骤3：验证安装
访问工时填报页面，如果看到左下角出现 **🚀 一键填写工时** 按钮，说明安装成功！

## 🎯 使用方法

### 界面布局
- **主按钮**：🚀 一键填写工时（左下角）
- **设置按钮**：⚙️（主按钮右侧）

### 方法一：快速填写（推荐）
1. 进入工时填报页面
2. **【重要】先手动选择工作日期**（页面上方的日期选择框）
3. 点击左下角的 **🚀 一键填写工时** 按钮
4. 脚本将自动填写第一行的所有字段：
   - ✅ 工作性质（科研工作）
   - ✅ 工作类别（根据工作性质自动匹配）
   - ✅ 内容属性（从模板中随机选择）
   - ✅ 工作形式（文字撰写）
   - ✅ 开始时间（2025-07-10 09:00）
   - ✅ 结束时间（2025-07-10 17:00）
   - ✅ 投入工时（系统自动计算）
   - ✅ 备注信息（工作正常完成）
   - 🔧 关联项目（需配置后自动选择）
   - 🔧 共同完成人（需配置后自动选择）

### 方法二：多行填写
1. 点击"增加"按钮添加新行
2. 点击 **🚀 一键填写工时** 按钮
3. 在弹出的对话框中选择要填写的行数
4. 脚本将按顺序填写指定数量的行

### 方法三：配置管理
1. 点击 ⚙️ 设置按钮打开配置面板
2. 可以：
   - 📝 编辑当前配置
   - ➕ 创建新配置
   - 📋 复制现有配置
   - 🗑️ 删除配置
   - 📤 导出配置到JSON文件
   - 📥 从JSON文件导入配置

## 🔧 时间格式与工时计算说明

### 时间格式转换
脚本会自动将简单时间格式转换为系统要求的完整格式：
- **输入格式**：09:00, 17:00
- **输出格式**：2025-07-10 09:00, 2025-07-10 17:00

### 日期来源说明
**重要更新**：脚本会智能读取页面上的工作日期：

1. **优先使用工作日期栏**：
   ```javascript
   // 脚本读取工作日期输入框的值
   const workDateInput = document.querySelector('input[name="workDate"]');
   const workDate = workDateInput.value; // 例如：2025-07-10
   ```

2. **自动兜底机制**：
   - 如果工作日期栏有值，使用该日期
   - 如果工作日期栏为空，使用当前日期作为备选
   - 确保日期校验能够通过

3. **日期一致性保障**：
   - 开始时间和结束时间使用相同的日期
   - 避免跨日期的时间冲突
   - 符合系统的日期校验规则

### 工时自动计算
**重要说明**：脚本不会手动填写工时数字，而是让系统自动计算：

1. **系统计算逻辑**：
   - 填写开始时间和结束时间后
   - 系统自动计算时间差
   - 向上取整为整小时数
   - 自动填入"投入工时"字段

2. **脚本处理方式**：
   ```javascript
   // 脚本填写开始和结束时间后
   fillFormField(baseSelectors.startTime, "2025-07-10 09:00");
   fillFormField(baseSelectors.endTime, "2025-07-10 17:00");
   
   // 方法1：模拟点击结束时间输入框打开日期选择器
   endTimeInput.click();
   
   // 查找并点击Layui日期选择器的"确定"按钮
   const confirmButton = document.querySelector('.laydate-btns-confirm');
   confirmButton.click(); // 触发工时自动计算
   
   // 方法2（备用）：直接触发change事件
   endTimeInput.dispatchEvent(new Event('change'));
   ```

3. **智能兜底机制**：
   - 🎯 **主要方法**：模拟点击Layui日期选择器的"确定"按钮
   - 🔄 **备用方法**：直接触发change事件
   - 🛡️ **兜底方案**：如果自动计算失败，使用配置的默认工时
   - ✅ **结果验证**：检查工时字段是否有值，提供明确反馈

4. **优势**：
   - ✅ 确保工时计算准确
   - ✅ 避免手动计算错误
   - ✅ 符合系统业务逻辑
   - ✅ 支持任意时间段计算
   - ✅ 多重保障机制

5. **示例**：
   | 开始时间 | 结束时间 | 系统计算工时 |
   |----------|----------|--------------|
   | 09:00 | 17:00 | 8小时 |
   | 13:30 | 16:00 | 3小时 |
   | 08:00 | 12:30 | 5小时 |

## 🚨 注意事项

### ⚠️ 重要提醒
1. **确保信息准确** - 脚本提高效率，但请确保填写的工时信息真实准确
2. **逐行填写** - 建议一行一行填写，避免数据混乱
3. **检查后提交** - 自动填写后请核对所有信息，特别是项目和共同完成人
4. **配置备份** - 重要配置请及时导出备份

### 🔐 安全说明
- 脚本只在指定的公司域名下运行
- 不会收集或传输任何个人信息  
- 所有操作都在本地浏览器中完成
- 配置信息存储在浏览器本地

### 🐛 故障排除

#### 脚本不工作
1. ✅ 检查是否在工时填报页面
2. ✅ 确认Tampermonkey扩展已启用
3. ✅ 按F12查看Console控制台错误信息
4. ✅ 刷新页面重新加载脚本

#### 常见问题
| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 按钮不显示 | 页面未完全加载 | 等待2-3秒或刷新页面 |
| 填写不完整 | 页面元素加载慢 | 再次点击按钮或等待加载 |
| 项目选择失败 | 关键词不匹配 | 检查项目关键词配置 |
| 共同完成人选择失败 | 姓名不匹配 | 检查人员姓名配置 |
| 时间格式错误 | 系统兼容性 | 脚本会自动处理，如仍有问题请检查控制台 |
| 日期校验失败 | 工作日期栏为空 | 手动填写工作日期后再使用脚本 |

#### 调试信息
脚本提供详细的控制台日志，包括：
- 🔍 元素查找过程
- ⚙️ 配置加载状态
- 📊 表格识别结果
- 🎯 项目/人员搜索过程
- ✅ 操作成功/失败状态
