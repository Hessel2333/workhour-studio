# Work Trail

一个面向项目管理、任务流转和工时填报场景的前端原型应用。

当前版本聚焦于团队协作中的几个核心视角：工作台总览、时间轴工时录入、项目池管理、任务分发与状态流转、执行统计，以及内置的使用说明页。项目以本地前端状态为中心，适合继续演进为 Web 应用或桌面应用原型。

## 功能概览

- 工作台总览：展示当前用户视角下的任务、时间安排与执行状态
- 日程与工时：按天或周查看时间轴，录入和调整时间块
- 项目管理：查看项目池、阶段信息与项目状态
- 任务流转：支持不同角色视角下的任务查看和分发
- 统计分析：基于当前数据生成执行进度与分析结果
- 说明中心：内置上手指南，帮助理解系统逻辑和页面关系
- 本地持久化：通过浏览器 `localStorage` 保存应用状态和主题偏好

## 技术栈

- React 19
- TypeScript 5
- Vite 7
- 原生 CSS

## 本地开发

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

默认开发地址：

```text
http://localhost:4173
```

### 构建生产版本

```bash
npm run build
```

### 本地预览构建结果

```bash
npm run preview
```

## 项目结构

```text
src/
  components/     页面视图与模块组件
  content/        内置说明文档内容
  data/           初始化示例数据
  lib/            统计、时间、存储等工具函数
  App.tsx         主应用状态与视图切换
  main.tsx        应用入口
  styles.css      全局样式
  types.ts        核心业务类型定义
```

## 当前实现特点

- 当前数据来自本地示例数据与前端交互结果
- 状态保存在浏览器本地，不依赖后端服务
- 适合继续扩展为真实协作产品的交互原型
- 如果要上线多用户版本，建议后续补充服务端、账号体系和数据库存储

## 后续可扩展方向

- 接入后端 API 与数据库，实现多人协作
- 增加登录、权限和组织空间
- 支持项目模板、审批流和报表导出
- 在保持现有前端代码基础上封装为桌面应用
