# 快速开始

这套系统不是单纯的工时填报表，而是把 `项目`、`任务`、`日程排期`、`时间块记录` 和 `统计分析` 串成一个闭环。

## 一条主链路

1. 主管或产品先创建项目，再创建任务并分发给执行人。
2. 执行人进入任务页，只看自己的任务。
3. 任务被拖入日程表后，形成明确的时间块排期。
4. 时间块保存后，会自动累计为任务的实际工时。
5. 统计页根据时间块、状态变化、返工和阻塞记录做分析。

## 推荐上手顺序

1. 从左下角切换身份，理解不同角色为什么看到不同页面。
2. 以主管身份进入任务页，先创建项目和任务。
3. 切换成研发身份，进入任务页查看“我的任务”。
4. 把任务拖进日程页，体验排期与时间块调整。
5. 最后打开统计页，看项目、返工、阻塞和负载是如何被自动汇总的。

## 为什么把日程表放在中间

日程表不是附属功能，它是任务从“被分配”到“真正开始执行”的中间层。没有排进日程的任务，往往说明它还没有真正进入执行状态。
